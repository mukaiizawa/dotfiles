# Code Style

- DO NOT use non-English text in identifiers, comments, or log messages.
- DO NOT introduce unnecessary complexity; As simple as possible, but no simpler.
- DO NOT implement code that violates the Single Responsibility Principle (SRP). Keep each module, class, and function focused on a single reason to change.
- DO NOT use blank lines inside function bodies. If removing blank lines hurts readability, split the function instead.
- DO NOT add extra whitespace only to align columns.
- DO NOT use meaningless names for classes, functions, or variables. Conventional short names, narrow scopes, and short-lived variables are acceptable.
- DO NOT define unnecessary temporary variables. Use them only when they clarify meaning.
- DO NOT ignore the local style of the existing file.
- DO NOT override the project's formatter, linter, or established conventions with these preferences.
